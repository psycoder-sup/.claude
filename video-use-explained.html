The LLM watches the video. It reads it — a text transcript from audio is always loaded; visual composites are pulled only at decision points.
Why it works
LAYER 1 — always loaded
Audio transcript

One ElevenLabs Scribe call per source → word-level timestamps, speaker diarization, and audio events like (laughter). Every take packs into a single ~12 KB takes_packed.md — the primary reading view.

LAYER 2 — on demand
Visual composite

timeline_view renders a filmstrip + waveform + word-label PNG for any range. Called only at decision points — ambiguous pauses, retake comparisons, cut-point sanity checks. Never a scan tool.

The pipeline
The session, step by step
1
Inventory
ffprobe every source, transcribe the directory, pack into takes_packed.md, sample a timeline_view or two.
2
Pre-scan for problems
One pass to note verbal slips, mis-speaks, phrasings to avoid. Feeds the editor brief.
3
Converse
Describe what you see, ask questions shaped by the material. No fixed checklist — the right questions differ every time.
4
Propose strategy · wait
4–8 sentences: shape, take choices, cut direction, animation + grade plan, length estimate. Wait for confirmation.
5
Execute
Produce edl.json, drill into timeline_view at ambiguous moments, build animations in parallel sub-agents, grade per-segment, compose with render.py.
6
Preview
render.py --preview — fast 720p pass.
7
Self-eval · before showing
timeline_view on the rendered output at every cut: discontinuity, audio pop, hidden subtitle, misaligned overlay. Fix → re-render, cap 3 passes.
8
Iterate + persist
Natural-language feedback, re-plan, re-render. Final on confirmation. Append the session to project.md. Never re-transcribe.

The 12 Hard Rules
non-negotiable

These aren't taste — they're production correctness. Deviation produces silent failures or broken output. Everything else in the skill is artistic freedom.

01Subtitles applied LAST in the filter chain, after every overlay.
02Per-segment extract → lossless -c copy concat, never single-pass filtergraph.
0330ms audio fades at every segment boundary — otherwise audible pops.
04Overlays use setpts=PTS-STARTPTS+T/TB so frame 0 lands at the window start.
05Master SRT uses output-timeline offsets — else captions misalign after concat.
06Never cut inside a word. Snap every edge to a transcript word boundary.
07Pad every cut edge (30–200ms) to absorb Scribe's 50–100ms drift.
08Word-level verbatim ASR only. Never SRT/phrase mode, never normalized fillers.
09Cache transcripts per source. Re-transcribe only if the source file changed.
10Parallel sub-agents for multiple animations. Spawn N at once, never sequential.
11Strategy confirmation before execution. Never touch the cut until the plan is approved.
12All session outputs in <videos_dir>/edit/. Never write inside the project dir.
Cut craft — audio-first, every cut must work on both tracks
Where cuts come from
  • • Candidates from word boundaries and silence gaps.
  • • Silences ≥ 400ms = cleanest cut targets.
  • • 150–400ms phrase boundaries = usable with a visual check.
  • • < 150ms = unsafe (mid-phrase).
Preserve the peaks
  • • Laughs, punchlines, emphasis beats — keep them.
  • • Extend past punchlines to include the reaction: the laugh is the beat.
  • • Audio events (laughs) (sighs) (applause) mark beats.
  • • Speaker handoffs want air: 400–600ms (taste call).
Creative layers — worked examples, not mandates
Animations
4 engines, pick per slot
  • HyperFrames — HTML/CSS/GSAP, UI motion, alpha WebM.
  • Remotion — React composition + component state.
  • Manim — diagrams, equations, graph morphs.
  • PIL + ffmpeg — counters, typewriter, simple cards.
One sub-agent = one animation = one file. Spawn them in parallel. Easing is always cubic — never linear.
Color grade
Reason about the image

Mental model is ASC CDL: out = (in·slope + offset)power, then saturation. Look at a frame, adjust one thing, look again.

  • • warm_cinematic — teal/orange, desaturated.
  • • neutral_punch — contrast + gentle S-curve.
  • • none — straight copy (default).
Applied per-segment during extraction, never post-concat.
Subtitles
Three dimensions
  • • Chunking — 1/2/3 words or sentence.
  • • Case — UPPER / Title / Natural.
  • • Placement — bottom margin.

Anti-patterns — things that consistently fail, regardless of style
× Pre-computed codec formats with tone/shot/usability tags — derive at decision time.
× Hand-tuned moment-scoring functions — the LLM picks better.
× Whisper SRT / phrase output — loses sub-second gap data.
× Whisper on CPU — slow, and it normalizes fillers.
× Single-pass filtergraph with overlays — double re-encodes.
× Linear animation easing — looks robotic.
× Typing text centered on the partial string — slides left as it grows.
× Sequential sub-agents for animations — always parallel.
× Editing before confirming the strategy — never.
× Assuming what kind of video it is — look, ask, then edit.
Audio transcript is the primary surface. • Visual composites on demand at decision points. • 12 hard rules, artistic freedom everywhere else.
